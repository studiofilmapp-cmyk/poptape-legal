1. FFmpeg / FFmpeg-kit (LGPL)

Moafilm uses FFmpeg and the FFmpeg-kit Flutter wrapper for video processing.

  • Package: ffmpeg_kit_flutter_new_full 2.1.0
  • FFmpeg version: 8.0.0 (non-GPL / LGPL build)
  • Wrapper: sk3llo/ffmpeg_kit_flutter (a fork of arthenica/ffmpeg-kit), LGPL v3.0

The build Moafilm uses is a non-GPL (LGPL) build that contains no GPL components. (GPL libraries such as x264, x265, xvidcore, and vid.stab are not included.)

LGPL Compliance

  • Modifications: Moafilm uses FFmpeg and FFmpeg-kit unmodified.
  • Dynamic linking: The FFmpeg libraries are embedded as dynamic frameworks (.framework), so users can replace or re-link the libraries.
  • Corresponding source: The exact source code for the version used is available below.

Source Code and Full License Text

  • FFmpeg 8.0 source: github.com/FFmpeg/FFmpeg (tag n8.0)
  • FFmpeg-kit wrapper / build scripts: github.com/sk3llo/ffmpeg_kit_flutter
  • LGPL v3.0 full text: gnu.org/licenses/lgpl-3.0.html
  • LGPL v2.1 full text: gnu.org/licenses/old-licenses/lgpl-2.1.html

External libraries bundled with FFmpeg

The FFmpeg full build includes the following libraries, each under a permissive or LGPL license: dav1d, opus, libvpx, libwebp, libvorbis, libtheora, snappy, speex (BSD) · fontconfig, libxml2 (MIT) · libass (ISC) · freetype (FTL) · fribidi, gmp, gnutls, lame, shine, soxr, twolame, libiconv (LGPL) · opencore-amr, vo-amrwbenc (Apache) · kvazaar (BSD) · zimg (permissive), and others.
